# 福小酿点赞计数按钮设计

## 目标

将现有带文字和冒泡效果的点赞按钮简化为清晰的点赞计数控件：左侧显示点赞图标，右侧显示全站总点赞数。点赞成功后通过图标状态与五秒 Toast 提供反馈。

## 视觉与状态

- 按钮只包含两个可见元素：左侧点赞图标、右侧全站总点赞数。
- 图标使用项目内联 SVG，不引入图标库。
- 未点赞时 SVG 仅显示描边，呈空心状态。
- 点赞成功后 SVG 使用当前强调色填充，呈实心状态。
- 请求进行中禁用按钮，避免重复提交。
- 已点赞用户继续看到实心图标和最新总数，按钮保持禁用。
- 总数尚未加载时右侧显示 `—`，不伪造数字。

## 成功与失败反馈

- 点赞成功后更新 D1 返回的总数，并将图标切换为实心。
- 同时显示现有顶部 Toast，文案固定为：`感谢！福小酿收到你的点赞啦。`
- 成功 Toast 显示五秒后自动消失。
- 删除当前按钮内部的多个 `👍` 冒泡动画与常驻感谢文字。
- 接口失败仍使用现有 Toast 提示：`福小酿暂时没接住这个赞`，且不更改图标、点赞状态和总数。

## 数据与统计

- 保留现有 `/api/likes` GET/POST 接口与 Cloudflare D1 全站累计逻辑。
- 保留浏览器本地防重复点赞记录。
- 保留 Plausible `Like Fuxiaoniang` 事件，仍只在成功点赞后发送。
- 本次不修改其他 Toast 的默认显示时长；`flash` 接受可选持续时间，只有点赞成功传入 `5000` 毫秒。
- Toast 定时器使用 ref 管理；连续提示时清除旧定时器，组件卸载时清理，避免旧定时器提前关闭新提示。

## 可访问性

- 按钮保留 `aria-pressed` 和禁用状态。
- 按钮增加随状态变化的 `aria-label`，包含是否已点赞和当前总点赞数。
- SVG 标记为装饰性，避免屏幕阅读器重复朗读。
- Toast 保留 `role="status"`。

## 验证

- 源码契约测试验证 SVG 空心/实心状态、图标与计数布局、成功 Toast 文案和 `5000` 毫秒时长。
- 测试确认旧的 `👍` 冒泡节点、常驻感谢文字和相关动画样式已删除。
