# 组合画像酒方名匹配设计

## 目标

保留现有全部动态酒方名，不增加、不删除、不改文案。酒方名不再由某一道题直接决定，而是根据五道题的完整答案组合匹配三个候选名。

## 生成逻辑

1. 继续使用现有 `ANSWER_WEIGHTS`，把五个答案汇总为完整人格维度。
2. 为每个现有酒方名配置一组隐藏风格权重，例如能量、松弛、社交、反思、行动、荒诞、AI 感。
3. 用完整人格维度对全部现有酒方名评分。
4. 按分数选出三个不同名称；若分数相同，使用完整答案与 seed 的稳定哈希排序。
5. 相同答案和 seed 必须稳定产生相同的三个名称。
6. 单独改变任何一道题都可能影响最终三个名称，但不存在“某答案固定对应某名称”的关系。

## 展示规则

- 结果页继续显示一个酒方名，并允许在三个候选名中循环切换。
- 保留现有 `1/3`、`2/3`、`3/3` 计数方式。
- 推荐酒款、配方、无酒精模式与今日状态逻辑不变。

## 边界处理

- 所有名称必须唯一，禁止同一结果出现重复名称。
- 画像分数接近时使用稳定哈希增加组合差异，避免大量组合得到同一组三个名字。
- 不调用在线 AI，不新增接口或费用。

## 测试

- 验证全部 4500 种答案组合始终返回三个不同名称。
- 验证相同答案与 seed 返回稳定结果。
- 验证改变单个答案时，存在可观察的候选名变化。
- 验证现有动态酒方名集合没有增加、删除或改名。
